## 2. The Fundamental Problem (Element Schism)

**There are TWO incompatible Element systems actively maintained:**

### 2.1 Hydrogen.Element.Core (897 lines)

**Location**: `src/Hydrogen/Element/Core.purs`

**What it claims to be**:
- "Pure data description of visual elements"
- "CORRECT Element type: pure data composed entirely from Schema atoms"
- "No strings. No DOM concepts. No CSS. No JavaScript."
- "Replaces the broken Hydrogen.Render.Element"
- "GPU-native instruction set"
- "Deterministic — same Element = same pixels"

**What it actually exports**:
- Element ADT with 11 variants: Rectangle, Ellipse, Path, Text, Image, Video, Audio, Model3D, Group, Transform, Empty
- Spec types: RectangleSpec, EllipseSpec, PathSpec, TextSpec, GlyphSpec, GroupSpec, TransformSpec, StrokeSpec
- Media types: ImageSpec, ImageSource, VideoSpec, VideoSource, VideoPlayback, AudioSpec, AudioSource, AudioPlayback, Model3DSpec, Model3DSource, Model3DCamera
- Smart constructors: rectangle, ellipse, path, text, image, video, audio, model3D, group, transform, empty
- Stroke constructors: stroke, strokeWith, noStroke
- Re-exports: LineCap(..), LineJoin(..)

**Critical problem**: NO EVENT HANDLING
- No `msg` type parameter
- No `onClick`, `onMouseDown`, `onHover` — nothing
- No `Attribute` type
- No `EventHandler` type

**Used by** (~20 imports):
- Element/Binary.purs (serialization)
- Element/Flatten.purs (DrawCommand conversion)
- Composition/Source.purs
- Composition/Bridge.purs

**Verdict**: Cannot build interactive UI. Shapes only.

### 2.2 Hydrogen.Render.Element (934 lines)

**Location**: `src/Hydrogen/Render/Element.purs`

**What it claims to be**:
- "Pure Data Description of UI"
- Can target: Halogen HTML, Direct DOM, Static HTML, Canvas, WebGL, Native views
- "Elements are pure, testable, serializable"

**What it actually exports**:

Core Types:
- Element(..) — ADT with 5 variants (Text, Element, Keyed, Lazy, Empty)
- Attribute(..) — ADT with 6 variants (Attr, AttrNS, Prop, PropBool, Handler, Style)
- EventHandler(..) — ADT with 31 event types
- Namespace(..) — HTML, SVG, MathML, Custom

Element Constructors (46):
- Generic: element, elementNS, text, empty, keyed, lazy
- HTML: div_, span_, p_, a_, button_, input_, img_, form_, label_, h1_-h6_, ul_, ol_, li_, table_, tr_, td_, th_, thead_, tbody_, tfoot_, caption_, section_, article_, header_, footer_, nav_, main_, aside_, strong_, em_, code_, pre_, blockquote_, hr_, br_, meta_, link_, script_, textarea_
- SVG: svg_, svgElement, circle_, rect_, path_, line_, g_, defs_, use_, clipPath_, mask_, linearGradient_, radialGradient_, stop_, text_, tspan_, polyline_

Attributes (36):
- attr, attrNS, prop, propBool, class_, classes, id_, style, styles, href, src, alt, title, placeholder, value, type_, name, min_, max_, step_, disabled, checked, selected, readonly, required, autofocus, tabIndex, role, ariaLabel, ariaHidden, ariaDescribedBy, ariaLabelledBy, ariaLive, ariaAtomic, ariaRole, dataAttr

Event Handlers (28):
- onClick, onDoubleClick, onMouseDown, onMouseUp, onMouseMove, onMouseEnter, onMouseLeave, onFocus, onBlur, onInput, onChange, onSubmit, onKeyDown, onKeyUp, onKeyPress, onScroll, onWheel, onDragStart, onDrag, onDragEnd, onDragEnter, onDragLeave, onDragOver, onDrop, onTouchStart, onTouchMove, onTouchEnd, onTouchCancel

Manipulation:
- mapMsg, filterChildren, prependChild, appendChild, withAttribute, withAttributes, forceLazy, forceAllLazy, childCount, firstChild, lastChild, isEmpty

**Critical problem**: UNBOUNDED STRINGS EVERYWHERE
- `tag :: String` — any string, not bounded
- `Attr String String` — name AND value unbounded
- `Style String String` — CSS property AND value unbounded
- `CustomEvent String msg` — event name unbounded

**Used by** (187 imports):
- The entire UI library
- All Element/Compound/* files
- All UI/* files

**Verdict**: Can build interactive UI but violates "no strings" principle.

### 2.3 The Schism

Element.Core line 51-56 explicitly states it "replaces the broken Hydrogen.Render.Element" — yet:
- Render.Element has 187 dependents
- Element.Core has ~20 dependents
- Both are actively maintained
- No migration path exists
- They are fundamentally incompatible types

**Nobody decided which one wins.**

## 3. Missing Interaction Primitives

### 3.1 MouseButton — EXISTS BUT WRONG LOCATION

**Found in**: `src/Hydrogen/GPU/FrameState.purs` (line 378)

**Should be in**: `src/Hydrogen/Schema/Gestural/Pointer.purs`

**Current definition**:
```purescript
data MouseButton
  = MouseLeft
  | MouseMiddle
  | MouseRight
  | MouseBack
  | MouseForward
```

**Problem**: This is a Schema atom (bounded enumeration of mouse buttons). It belongs in the Gestural pillar, not buried in GPU/FrameState.

### 3.3 What Schema/Gestural is MISSING

**MouseEvent type** — The actual event that occurs:
```purescript
-- DOES NOT EXIST
data MouseEventType
  = Click
  | DblClick
  | MouseDown
  | MouseUp
  | MouseMove
  | MouseEnter
  | MouseLeave
  | ContextMenu
  | AuxClick
  | Wheel

type MouseEvent =
  { eventType :: MouseEventType
  , button :: MouseButton
  , buttons :: Array MouseButton
  , position :: PointerPosition
  , modifiers :: ModifierState
  , timestamp :: Timestamp
  }
```

**PointerEvent type** — W3C Pointer Events (unifies mouse/touch/pen):
```purescript
-- DOES NOT EXIST
data PointerEventType
  = PointerDown
  | PointerUp
  | PointerMove
  | PointerEnter
  | PointerLeave
  | PointerOver
  | PointerOut
  | PointerCancel
  | GotPointerCapture
  | LostPointerCapture

type PointerEvent =
  { eventType :: PointerEventType
  , pointerId :: PointerId
  , pointerType :: PointerType
  , state :: PointerState
  , isPrimary :: Boolean
  }
```

**Unified GesturalEvent type**:
```purescript
-- DOES NOT EXIST
data GesturalEvent
  = PointerEvt PointerEvent

  | KeyboardEvt KeyboardEvent
  | TouchEvt TouchEvent
  | GestureEvt GestureEvent
```

**ComponentShortcuts** — Per-component keyboard shortcut definitions:
```purescript
-- DOES NOT EXIST
type ComponentShortcuts msg =
  { shortcuts :: Array { shortcut :: Shortcut, action :: msg, description :: String }
  , keyboardNav :: Maybe KeyboardNavConfig
  }
```

**InteractionSpec** — Complete interaction specification for a component:
```purescript
-- DOES NOT EXIST
type InteractionSpec msg =
  { onClick :: Maybe (MouseEvent -> msg)
  , onRightClick :: Maybe (MouseEvent -> msg)
  , onMiddleClick :: Maybe (MouseEvent -> msg)
  , onDoubleClick :: Maybe (MouseEvent -> msg)
  , onHover :: Maybe (HoverPhase -> msg)
  , onFocus :: Maybe (FocusEvent -> msg)
  , onKeyDown :: Maybe (KeyEvent -> msg)
  , shortcuts :: Array (Tuple Shortcut msg)
  , contextMenu :: Maybe (Array MenuItem)
  }
```

### 3.4 The Fundamental Gap

**Schema has**:
- STATE types (HoverState, DragState, FocusState, etc.)
- TRIGGER types (ContextTrigger, Shortcut, etc.)

**Schema is MISSING**:
- EVENT types (the actual thing that happens)
- UNIFIED event vocabulary

**Element.Core has**:
- SHAPE types (Rectangle, Path, etc.)

**Element.Core is MISSING**:
- INTERACTIVITY (how shapes respond to events)
- `msg` type parameter

**Render.Element has**:
- INTERACTIVITY (onClick, onKeyDown, etc.)

**Render.Element is MISSING**:
- TYPE SAFETY (strings everywhere)
- SCHEMA INTEGRATION (doesn't use Schema atoms)
